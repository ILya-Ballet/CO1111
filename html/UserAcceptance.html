Nielsen's Heuristics done by Theocharis Tsekouras

1) Visibility of system status

Buttons change color when being hovered or pressed

All pages have the same navigation bar making the use of the page easier for the user

Clear display of current page that the user is in making it easier to navigate


2) Match between system and the real world

All information and information on how to play the game is written in simple text easier to understand for a non programmer

3)User control and freedom

4) Consistency and standards

All pages follow the same design with the same navigation bar

5) Error prevention

Error message pops up when user tried to play the treasure hunt without inputing a name

6)Recognition rather than recall

Clean and simple design makes the treasure hunt easy to navigate/use for begginer users

Simple buttons etc. minimizing the user's memory load

7) Flexibility and efficiency of use

Our application allows experienced and inexperienced user to use our application

8) Aesthetic and minimalistic design

Dark colors smoother to the eye making up for a more pleasureable experience

Minimalistic not complicated design easier to navigate through

Nice background along with hoverable buttons for a more attractive looking website

9)Help users recognize , diagnose,and recover from errors

Error messages written in simple text and not in programming language easier for the user to understand what error he made

10)Help and documentation

No extra information about how to play due to simple design
